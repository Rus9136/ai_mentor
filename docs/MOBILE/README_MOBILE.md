## Что такое AI Mentor?

**AI Mentor** — адаптивная образовательная платформа для школьников 7-11 классов Казахстана. Платформа автоматически подстраивается под уровень знаний каждого ученика и помогает учителям отслеживать прогресс класса.

**Ключевая идея:** Каждый ученик учится в своём темпе. Система определяет уровень владения материалом (A, B или C) и адаптирует объяснения и сложность под конкретного ученика.

## Структура учебного контента

### Иерархия материалов

```
Предмет (Математика, История, Физика...)
    │
    └── Учебник (Алгебра 9 класс)
            │
            └── Глава (Квадратные уравнения)
                    │
                    └── Параграф (Формула дискриминанта)
                            │
                            ├── Текст объяснения
                            ├── Аудио версия
                            ├── Видео урок
                            ├── Флеш-карточки
                            ├── Встроенные вопросы
                            └── Тесты
```

### Два типа контента

| Тип | Кто создаёт | Кто видит |
|-----|-------------|-----------|
| **Глобальный** | Супер-администратор | Все школы |
| **Школьный** | Школьный администратор | Только своя школа |

Школьный администратор может **скопировать** глобальный учебник и изменить его под нужды своей школы (например, добавить дополнительные примеры или убрать сложные темы).

## Система уровней мастерства (A/B/C)

### Что это?

Каждый ученик получает уровень владения материалом по каждой главе:

| Уровень | Баллы | Что значит | Цвет |
|---------|-------|------------|------|
| **A** | 85-100% | Отлично понимает материал | Зелёный |
| **B** | 60-84% | Хорошо понимает, есть небольшие пробелы | Жёлтый |
| **C** | 0-59% | Нужна дополнительная помощь | Красный |

### Как рассчитывается?

Система анализирует последние 5 попыток прохождения тестов:
- Более новые результаты важнее старых
- Учитывается стабильность (не скачет ли результат)
- Учитывается тренд (улучшается или ухудшается)

**Пример:**
- Ученик набрал 90%, 88%, 92%, 85%, 87% — уровень **A** (стабильно высокие результаты)
- Ученик набрал 70%, 75%, 80%, 85%, 88% — уровень **B** → может перейти в **A** (растущий тренд)
- Ученик набрал 80%, 65%, 55%, 45%, 40% — уровень **C** (падающий тренд)

### Зачем это нужно?

1. **Ученику:** Понимает свой прогресс, видит над чем нужно работать
2. **Учителю:** Видит распределение класса, знает кому нужна помощь
3. **AI-помощнику:** Адаптирует объяснения под уровень ученика

## Путь ученика (Learning Flow)

### Шаги изучения параграфа

```
┌─────────┐    ┌─────────┐    ┌──────────┐    ┌─────────┐    ┌───────────┐
│  INTRO  │ →  │ CONTENT │ →  │ PRACTICE │ →  │ SUMMARY │ →  │ COMPLETED │
│         │    │         │    │          │    │         │    │           │
│Введение │    │ Изучение│    │ Практика │    │  Итог   │    │ Завершено │
└─────────┘    └─────────┘    └──────────┘    └─────────┘    └───────────┘
```

### Детали каждого шага

1. **INTRO** — Краткое введение: о чём этот параграф, что узнаете
2. **CONTENT** — Основной материал:
   - Текст с формулами и изображениями
   - Аудио версия (для тех, кто лучше воспринимает на слух)
   - Видео урок
   - Флеш-карточки для запоминания
3. **PRACTICE** — Встроенные вопросы для проверки понимания
4. **SUMMARY** — Итог + самооценка ученика:
   - "Всё понял" ✓
   - "Есть вопросы" ?
   - "Нужна помощь" ✗
5. **COMPLETED** — Параграф изучен, можно переходить к следующему

---

## Типы тестов

### По назначению

| Тип | Когда используется | Влияет на уровень? |
|-----|-------------------|-------------------|
| **Формативный** | После каждого параграфа | Да |
| **Суммативный** | После главы (итоговый) | Да |
| **Практический** | Для самопроверки | Нет |
| **Диагностический** | В начале темы | Нет |

### Типы вопросов

| Тип | Описание | Пример |
|-----|----------|--------|
| **Один ответ** | Выбрать один правильный из нескольких | Сколько будет 2+2? ○4 ○5 ○6 |
| **Несколько ответов** | Выбрать все правильные | Какие числа чётные? ☐2 ☐3 ☐4 |
| **Верно/Неверно** | Согласиться или опровергнуть | Земля круглая — Верно/Неверно |
| **Короткий ответ** | Ввести ответ текстом | Столица Казахстана: _____ |

### Процесс прохождения теста

```
1. Ученик нажимает "Начать тест"
   └── Создаётся попытка (attempt) с таймером

2. Отвечает на вопросы
   └── Можно отвечать по одному или все сразу

3. Нажимает "Завершить"
   └── Система проверяет ответы
   └── Рассчитывается балл
   └── Обновляется уровень мастерства

4. Смотрит результаты
   └── Какие вопросы правильные/неправильные
   └── Объяснения к ответам
```

## Домашние задания

### Как это работает

```
Учитель                              Ученик
   │                                    │
   │  1. Создаёт ДЗ                     │
   │     • Выбирает параграфы           │
   │     • Добавляет задания            │
   │     • Устанавливает дедлайн        │
   │                                    │
   │  2. Публикует ──────────────────►  │  3. Получает ДЗ
   │                                    │     • Видит задания
   │                                    │     • Видит дедлайн
   │                                    │
   │                                    │  4. Выполняет
   │                                    │     • Отвечает на вопросы
   │                                    │     • Получает подсказки от AI
   │                                    │
   │  6. Проверяет ◄────────────────── │  5. Сдаёт
   │     • Смотрит ответы               │
   │     • Комментирует                 │
   │     • Ставит оценку                │
   │                                    │
```

### Типы заданий в ДЗ

- **Quiz** — тест по параграфу
- **Reading** — прочитать и ответить на вопросы
- **Practice** — решить задачи

### Особенности

- Можно несколько попыток (учитель настраивает)
- Штраф за опоздание (например, -10% после дедлайна)
- AI помогает с объяснениями, но не даёт готовые ответы

## AI-помощник

### Возможности

| Функция | Описание |
|---------|----------|
| **Объяснение** | Простыми словами объясняет сложные темы |
| **Примеры** | Приводит дополнительные примеры |
| **Подсказки** | Направляет к решению, не давая готовый ответ |
| **Адаптация** | Меняет стиль в зависимости от уровня ученика |

### Адаптация под уровень

| Уровень | Как объясняет AI |
|---------|------------------|
| **A** | Кратко и по делу, сложные примеры |
| **B** | Подробнее, с пошаговым разбором |
| **C** | Максимально просто, базовые примеры, повторение основ |

### Типы сессий чата

- **reading_help** — помощь во время чтения параграфа
- **post_paragraph** — вопросы после изучения
- **test_help** — помощь во время теста
- **general_tutor** — общие вопросы по предмету

## ГОСО — Государственный стандарт образования

### Что это?

ГОСО определяет, что должен знать и уметь ученик каждого класса по каждому предмету в Казахстане.

### Структура

```
Предмет (История Казахстана)
    │
    └── Рамка ГОСО (версия 2023)
            │
            └── Раздел (Древний Казахстан)
                    │
                    └── Подраздел (Эпоха бронзы)
                            │
                            └── Учебная цель (7.2.1.2)
                                    │
                                    └── Связь с параграфами учебника
```

### Зачем это нужно?

1. **Контроль качества:** Учебный контент соответствует государственным требованиям
2. **Отчётность:** Учитель видит, какие цели ГОСО покрыты
3. **Планирование:** Можно увидеть пробелы в программе

## Школы и изоляция данных

### Принцип

Каждая школа видит **только свои данные**. Ученик школы №1 не может увидеть учеников или контент школы №2.

### Как это работает

```
┌─────────────────┐     ┌─────────────────┐
│    Школа №1     │     │    Школа №2     │
├─────────────────┤     ├─────────────────┤
│ Ученики школы 1 │     │ Ученики школы 2 │
│ Учителя школы 1 │     │ Учителя школы 2 │
│ Классы школы 1  │     │ Классы школы 2  │
│ Тесты школы 1   │     │ Тесты школы 2   │
└────────┬────────┘     └────────┬────────┘
         │                       │
         │   ┌───────────────┐   │
         └──►│ Глобальный    │◄──┘
             │ контент       │
             │ (учебники)    │
             └───────────────┘
```

### Исключение

**Глобальный контент** (созданный Супер-админом) виден всем школам, но редактировать его могут только Супер-админы.

## Синхронизация и офлайн-режим

### Идея

Ученик может учиться без интернета. Данные сохраняются локально и синхронизируются когда появится связь.

### Что работает офлайн

| Функция | Офлайн | Примечание |
|---------|--------|------------|
| Чтение учебников | Да | Если скачаны заранее |
| Прохождение тестов | Да | Результаты отправятся позже |
| Домашние задания | Да | Ответы сохраняются локально |
| AI-помощник | Нет | Требует подключения к серверу |

### Приоритет синхронизации

1. **Высокий:** Профиль, результаты тестов, домашние задания
2. **Средний:** Прогресс, уровни мастерства
3. **Низкий:** Контент учебников (большой объём)

## Авторизация и онбординг

### Способы входа

| Приложение | Email/Password | Google OAuth |
|------------|----------------|--------------|
| Student App | Нет | Да |
| Teacher App | Да | Да |
| Admin Panel | Да | Нет |

### Процесс регистрации ученика

```
1. Ученик нажимает "Войти через Google"
   │
2. Вводит код школы (получает от учителя)
   │
3. Заполняет профиль:
   • Имя, Фамилия
   • Класс
   │

```

### Код школы

Каждая школа имеет уникальный код (например, `SCHOOL001`). Этот код привязывает ученика к школе при регистрации.
