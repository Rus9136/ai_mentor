# AI Memory — Память ИИ-репетитора

> **Статус:** MVP (реализован 08.03.2026)

## Обзор

ИИ-репетитор AI Mentor теперь **помнит каждого ученика** между сессиями. Система памяти состоит из 3 слоёв, которые работают вместе для персонализации общения.

## Архитектура: 3 слоя памяти

```
┌─────────────────────────────────────────────────┐
│                 System Prompt                    │
│                                                  │
│  Базовый промпт (по типу сессии + mastery A/B/C)│
│  + Профиль ученика (long-term memory)            │
│  + Сводки последних 5 сессий (session memory)    │
│  + Контекст параграфа/теста                      │
│  + RAG-контекст из учебника                      │
│                                                  │
│  ┌───────────────────────────────────────────┐   │
│  │        Последние 10 сообщений             │   │
│  │         (short-term memory)               │   │
│  └───────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

### Слой 1: Short-term memory (был до этого)

Последние 10 сообщений текущей сессии. Позволяет LLM держать контекст диалога.

- **Хранение:** `chat_messages` (PostgreSQL)
- **Лимит:** `MAX_HISTORY_MESSAGES = 10`
- **Загрузка:** автоматически в `_build_llm_messages()`

### Слой 2: Session memory (новый)

После каждого разговора (4+ сообщений) LLM генерирует краткую сводку: тема, что ученик понял/не понял, уровень уверенности.

- **Хранение:** таблица `chat_session_summaries`
- **Генерация:** фоновая задача после каждого ответа чата
- **В промпт:** последние 5 сводок

**Пример записи:**
```
Тема: Квадратные уравнения
Сводка: Ученик затруднялся с дискриминантом, не мог определить
        количество корней. После пошагового объяснения понял формулу.
Пробелы: ["дискриминант", "количество корней уравнения"]
Уверенность: low
```

### Слой 3: Long-term memory (новый)

Структурированный профиль ученика. Обновляется инкрементально после каждой сессии — LLM анализирует новую сводку и дополняет существующий профиль.

- **Хранение:** таблица `student_memory`, одна строка на ученика (upsert)
- **Формат:** JSONB с полями:

```json
{
  "learning_style": "визуальный, предпочитает примеры",
  "weak_topics": ["квадратные уравнения", "логарифмы"],
  "strong_topics": ["линейные уравнения", "базовая алгебра"],
  "preferred_explanation_style": "пошаговые объяснения с аналогиями",
  "misconceptions": ["считает что деление на ноль равно нулю"],
  "interests": ["физика", "программирование"],
  "notes": "хорошо реагирует на поддержку, затрудняется с текстовыми задачами"
}
```

## Как это работает

### Поток данных

```
Ученик пишет сообщение
        │
        ▼
┌──────────────────────┐
│   ChatService        │
│                      │
│  1. Загрузить память │◄── student_memory + chat_session_summaries
│  2. Построить промпт │
│  3. RAG-поиск        │
│  4. LLM ответ        │
│  5. Сохранить         │
│  6. Вернуть ответ    │
└──────────┬───────────┘
           │
           ▼ (фоновая задача, не блокирует ответ)
┌──────────────────────────────────┐
│   StudentMemoryService           │
│                                  │
│  1. Суммаризация сессии (LLM)   │──► chat_session_summaries
│  2. Обновление профиля (LLM)    │──► student_memory
└──────────────────────────────────┘
```

### Что видит LLM в промпте

```
--- ПРОФИЛЬ УЧЕНИКА ---
Стиль обучения: визуальный, предпочитает примеры
Сильные темы: линейные уравнения
Слабые темы: квадратные уравнения, логарифмы
Заблуждения: считает что деление на ноль равно нулю
Предпочтения: пошаговые объяснения с аналогиями
Интересы: физика, программирование

ПРЕДЫДУЩИЕ СЕССИИ:
1. [07.03] Квадратные уравнения — затруднялся с дискриминантом (уверенность: низкая)
2. [05.03] Линейные уравнения — хорошо освоил тему (уверенность: высокая)
3. [03.03] Системы уравнений — понял метод подстановки, затруднялся с графическим (уверенность: средняя)

ИНСТРУКЦИЯ: Используй информацию для персонализации. Не упоминай ученику что у тебя есть память о нём.
```

### Graceful degradation

- **Первый разговор:** памяти нет → чат работает как раньше, без изменений
- **Короткий разговор (< 4 сообщений):** сводка не создаётся
- **Ошибка LLM при экстракции:** логируется, ответ чата не затрагивается
- **Ошибка парсинга JSON:** пропускается, повторится в следующей сессии

## Технические детали

### Таблицы БД

| Таблица | Назначение | Кардинальность |
|---------|------------|----------------|
| `chat_session_summaries` | Сводка каждой сессии | 1 на сессию |
| `student_memory` | Профиль ученика | 1 на ученика |

Обе таблицы: RLS по `school_id`, GRANT для `ai_mentor_app`.

### Файлы

| Файл | Назначение |
|------|------------|
| `backend/app/services/student_memory_service.py` | Основной сервис (загрузка, суммаризация, обновление) |
| `backend/app/models/student_memory.py` | Модели `ChatSessionSummary`, `StudentMemory` |
| `backend/alembic/versions/040_student_memory.py` | Миграция |

### Токен-бюджет

| Компонент | Токенов |
|-----------|---------|
| Базовый промпт | ~100-200 |
| Профиль ученика | ~200 max |
| Сводки сессий (5 шт) | ~500 max |
| Контекст параграфа/теста | ~500-1500 |
| RAG-контекст | ~1000 |
| История сообщений (10 шт) | ~2000-3000 |
| **Итого** | **~4000-5500** |

Бюджет памяти ограничен ~800 токенами (~3200 символов). При превышении текст обрезается.

### Стоимость

Каждая сессия генерирует 2 дополнительных LLM-вызова (суммаризация + обновление профиля). При использовании Cerebras (~$0.50/1M токенов) это ~$0.001 за сессию.

## Teacher Memory (память учительского чата)

Та же 3-слойная архитектура, адаптированная для учителя.

### Отличия от студенческой памяти

| Аспект | Студент | Учитель |
|--------|---------|---------|
| **Фокус профиля** | Пробелы, стиль обучения, заблуждения | Стиль преподавания, частые темы, методические предпочтения |
| **Сводка сессии** | knowledge_gaps, confidence_level | key_insights, session_type |
| **Роли в диалоге** | "Ученик" / "Репетитор" | "Учитель" / "Ассистент" |

### Профиль учителя (memory_data)

```json
{
  "teaching_style": "предпочитает пошаговые объяснения с аналогиями",
  "frequent_topics": ["квадратные уравнения", "дискриминант"],
  "preferred_preparation_style": "ищет примеры и задачи для учеников",
  "methodological_preferences": ["дифференцированный подход"],
  "subject_expertise": ["алгебра", "геометрия"],
  "common_requests": ["генерация контрольных", "объяснения для слабых"],
  "notes": "часто готовится к урокам алгебры 9 класс"
}
```

### Таблицы БД

| Таблица | Назначение | Кардинальность |
|---------|------------|----------------|
| `teacher_chat_session_summaries` | Сводка каждой сессии | 1 на сессию |
| `teacher_memory` | Профиль учителя | 1 на учителя |

### Файлы

| Файл | Назначение |
|------|------------|
| `backend/app/services/teacher_memory_service.py` | Основной сервис (загрузка, суммаризация, обновление) |
| `backend/app/models/teacher_memory.py` | Модели `TeacherChatSessionSummary`, `TeacherMemory` |
| `backend/alembic/versions/043_teacher_memory.py` | Миграция |

## Что нужно доработать (roadmap)

### Приоритет 1 — Улучшение текущего MVP

- [ ] **Учёт предмета/учебника в памяти** — сейчас профиль общий, нужно разделение по предметам (алгебра vs история)
- [ ] **Очистка устаревших данных** — автоматическое удаление сводок старше 3 месяцев
- [ ] **Ограничение частоты экстракции** — не запускать экстракцию на каждое сообщение, а только когда сессия "завершена" (пауза 10+ минут или 10+ сообщений)
- [ ] **Retry при ошибках парсинга JSON** — повторная попытка с другим промптом

### Приоритет 2 — Расширение функциональности

- [ ] **Knowledge gap tracking по компетенциям ГОСО** — привязка пробелов к конкретным `learning_outcomes`, а не к текстовым описаниям тем
- [ ] **Дашборд учителя** — отображение профилей учеников, их пробелов и прогресса в Teacher App
- [ ] **API для чтения/сброса памяти** — эндпоинт для ученика, чтобы видеть что ИИ о нём знает, и возможность сброса
- [ ] **Рекомендации параграфов** — на основе `weak_topics` рекомендовать ученику конкретные параграфы для повторения

### Приоритет 3 — Продвинутая память

- [ ] **Vector memory** — хранение embeddings прошлых разговоров для семантического поиска ("когда мы обсуждали дроби?")
- [ ] **Behavioral signals** — отслеживание паттернов: время ответа, длина сообщений, частота использования, время суток
- [ ] **Мультимодальная память** — учёт результатов тестов, домашних заданий, времени на параграфах для более точного профиля
- [ ] **Межпредметные связи** — если ученик силён в алгебре, использовать математические аналогии при объяснении физики
- [ ] **Настройки ученика** — явный выбор: "объясняй кратко / подробно", "используй казахский / русский", "давай больше примеров"
