# Спиральное обучение и граф зависимостей

## Зачем нужны связи между параграфами

В казахстанской системе образования используется **спиральное обучение**: одни и те же темы изучаются каждый год с нарастающей глубиной. Например, "Қазақ хандығы" проходится в 6, 7, 10 и 11 классах — каждый раз глубже.

**Граф зависимостей (prerequisites)** — это направленные связи между параграфами: "для понимания параграфа X нужно знать параграф Y".

### Ключевая цель

Эти связи — основа для **индивидуальной траектории обучения**:

1. Ученик не понимает тему → система находит prerequisite с низким effective_score
2. Система предлагает: "Сначала повтори §Y из N-класса"
3. Ученик закрепляет базу → возвращается к текущей теме
4. Формируется персональный маршрут обучения

### Типы связей

| Strength | Описание | Поведение |
|----------|----------|-----------|
| `required` | Обязательный пререквизит. Без него тему понять крайне сложно | `can_proceed: false` при effective_score < 60% |
| `recommended` | Рекомендуемый. Помогает, но не критично | Предупреждение, `can_proceed: true` |

### Порог (threshold)

`effective_score >= 0.60` — пререквизит считается выполненным. Используется effective_score с time decay, не best_score.

## Паттерны связей по предметам

### Алгебра (subject_id=24)

**Тип спирали: линейное наращивание сложности**

Темы строятся строго последовательно. Каждый класс вводит новые концепции, опираясь на предыдущие:

- **Степени** (7кл) → **Корни** (8кл) → **n-степени корень** (11кл)
- **Многочлены** (7кл) → **Квадрат теңдеу** (8кл) → **Многочлены высших степеней** (10кл)
- **y=ax²** (7кл) → **y=ax²+bx+c** (8кл) → **Функция углублённо** (10кл)
- **Тригонометрия** (9кл) → **Триг. функции** (10кл) → **Триг. уравнения** (10кл)
- **Пределы** (10кл) → **Производная** (10кл) → **Интеграл** (11кл) → **Дифф. уравнения** (11кл)
- **Комбинаторика** (9кл) → **Комбинаторика** (10кл)
- **Вероятность** (9кл) → **Вероятность** (10кл) → **Случайные величины** (10кл)

**Правила для алгебры:**
- Связи обычно `required` (математика строго последовательна)
- `recommended` только для повторительных/обзорных тем
- Внутри учебника тоже бывают важные зависимости (пределы → производная)

### История Казахстана (subject_id=1)

**Тип спирали: хронологическая + тематический синтез**

Классы 5-9 идут хронологически:
- 5 кл: Ежелгі дәуір (Древность)
- 6 кл: Орта ғасырлар (Средневековье)
- 7 кл: XVIII-XIX ғғ.
- 8 кл: XX ғ. первая половина
- 9 кл: 1945 — сегодня

Классы 10-11 — **тематический синтез** всех периодов (обобщение).

**Правила для истории:**
- 10-11 классы обобщают → связывать с конкретными параграфами из 5-9 кл
- Связи обычно `required` для тем, которые напрямую обобщаются
- `recommended` для косвенно связанных тем

### Информатика (subject_id — уточнить)

**Тип спирали: тематические блоки с углублением**

Блоки: программирование, алгоритмы, информационная безопасность, компьютерные сети, базы данных — повторяются из года в год с углублением.

**Правила для информатики:**
- Связи в основном между одноимёнными тематическими блоками разных классов
- `required` для продолжения одного блока
- `recommended` для смежных блоков

### Английский язык (Spotlight)

**Тип спирали: грамматика + лексика по темам**

Грамматические конструкции повторяются и усложняются. Лексические темы (Family, Travel, Science) возвращаются.

**Правила для английского:**
- Связи по грамматическим темам: Present Simple (5кл) → Present Perfect (7кл)
- Связи по секциям: Reading, Use of English повторяются в каждом модуле
- Связи в основном `recommended` (язык менее строго последователен)

## Как определять связи

### Алгоритм для агента

1. **Выгрузить все параграфы предмета** — группировать по учебнику (классу)
2. **Найти тематические кластеры** — параграфы с похожими названиями/темами через классы
3. **Для каждого кластера определить порядок** — от простого к сложному (по классу)
4. **Создать связи** — младший класс → старший класс
5. **При неясности — прочитать содержимое** параграфа для уточнения

### Признаки тематической связи

- Совпадение ключевых слов в названиях (стемминг/подстрока)
- Один исторический период или персонаж
- Одна математическая концепция на разных уровнях сложности
- Один грамматический паттерн

### Определение strength

- `required`: тема B невозможна без темы A (новая формула использует предыдущую)
- `recommended`: тема B полезна с контекстом A, но можно понять и без неё
